# Протокол OSPF
OSPF — протокол динамической маршрутизации, использующий концепцию разделения на области в целях масштабирования
## Задание
* Поднять три виртуалки </br>
* Объединить их разными vlan </br>
* поднять OSPF между машинами на базе Quagga; </br>
* изобразить ассиметричный роутинг; </br>
* сделать один из линков "дорогим", но что бы при этом роутинг был симметричным. </br>

- Устанавливаем пакет frr, работающий на базе Quagga </br>
- включаем forwarding </br>
- в файле /etc/frr/daemons включаем протокол ospf = yes. </br>
- выполняются настройки ospf через файл /etc/frr/frr.conf </br>
*!Указываются сети, которые анонсиуются соседним роутерам (Пример router1)* </br>
 network 10.0.10.0/30 area 0 </br>
 network 10.0.12.0/30 area 0 </br>
 network 192.168.10.0/24 area 0 </br>
*!Указываем адрес log-файла </br>*

log file /var/log/frr/frr.log </br>
default-information originate always </br>
!
