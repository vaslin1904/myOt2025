## Написать свою реализацию ps ax используя анализ /proc
ps ax — команда, которая выводит список текущих запущенных процессов в системе, включая демонов (процессы в фоновом режиме). В выводе есть столбцы: 

**PID** — идентификатор процесса (уникальный номер, назначенный каждому процессу ядром);
**TTY** — управляющий терминал (знак «?» указывает, что у процесса нет управляющего терминала);
**STAT** — состояние процесса (например, S (спящий), R (запущенный), Z (зомби) и т. д.);
**TIME** — время, в течение которого процесс был активен;
**COMMAND** — команда, которая запустила процесс.

Скрипт [script](https://github.com/vaslin1904/myOt2025/blob/main/work10Proc/ps_ax.sh)
Для всех процессов в /proc читаем в stat данные:
comm state ppid pgrp session tty_nr tpgid flags minflt cminflt majflt cmajflt utime stime cutime cstime priority nice num_threads itrealvalue starttime vsize rss
Преобразуем время выполнения процесса (utime + stime)
Определяем TTY (терминал)
Определяем состояние процесса (STAT)
Читаем команду (COMMAND)
Выводим информацию в формате ps ax
Для выполнения скрипта даем ему права на исполнение: **sudo chmod +x ps_ax.sh**
